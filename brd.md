 Business Requirements Document (BRD)

 Project: Box Cricket Turf Booking System



1. Objective

Build a web application that allows users to easily book cricket turf slots online and make advance payments, reducing manual booking effort for the owner.



 2. Business Goals

* Increase turf bookings
* Reduce phone/manual booking
* Provide better customer experience
* Enable online payments



 3. Target Users

* Local players (age 16–40)
* Groups booking turf for matches
* Mobile-first users


 4. Core Features (Phase 1 - MVP)

 4.1 Turf Slot Booking

* View available slots by date
* Select time slot
* Prevent double booking

4.2 Booking Flow

* Enter name and phone number
* Confirm booking

 4.3 Payment Integration

* Advance payment via Razorpay
* Booking confirmed after payment

 4.4 Admin Panel

* View all bookings
* Block/unblock slots
* Manage pricing

---

5. User Flow

1. User visits website
2. Clicks "Book Now"
3. Selects date & time slot
4. Enters details
5. Makes payment
6. Receives confirmation

---

6. Non-Functional Requirements

* Mobile-friendly design
* Fast loading (<3 seconds)
* Simple and intuitive UI
* Secure payment handling

---
 7. Future Scope (Phase 2)

* Restaurant table booking
* Food ordering system
* Offers & discounts
* WhatsApp notifications

---

 8. Success Criteria

* User can book a slot in under 30 seconds
* No double bookings
* Owner can manage bookings easily
